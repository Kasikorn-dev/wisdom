# Wisdom Phase 1 — Product Specification

> บันทึกเมื่อ: มีนาคม 2026  
> AI Provider: Anthropic

---

## 1. ภาพรวม Phase 1

Phase 1 ของ Wisdom โฟกัส 8 features หลัก:

1. **Auth** — Login (email + Continue with Google), Sign up, Forgot password, Reset password
2. **Personalization** — ผู้ใช้พิมพ์บอกว่าอยากเรียนอะไร เป้าหมายคืออะไร
3. **Accuracy** — เนื้อหาที่ gen ออกต้องถูกต้องตามหลักวิชาการ
4. **Active** — เกมแบบ active recall (quiz, flashcard)
5. **Privacy** — ข้อมูลผู้ใช้เก็บเป็นความลับ
6. **Safety** — พื้นที่ปลอดภัย ลองผิดลองถูกได้
7. **ความชัดเจนเรื่องความถูกต้อง** — นโยบายและ guardrails สำหรับเนื้อหา
8. **AI Guardrails + Fallback** — ป้องกันและจัดการเมื่อ AI error

---

## 2. Auth

Authentication ใช้ **Supabase Auth** (ตาม tech stack ของโปรเจค)

### 2.1 ฟีเจอร์ที่ต้องมี

| ฟีเจอร์ | รายละเอียด |
|---------|-------------|
| **Sign up** | สมัครสมาชิกด้วย email + password |
| **Login** | เข้าสู่ระบบด้วย email + password หรือ **Continue with Google** |
| **Continue with Google** | ลงชื่อเข้าใช้/สมัครด้วยบัญชี Google (OAuth) — ปุ่มเดียวใช้ได้ทั้ง login และ sign up |
| **Forgot password** | ขอลิงก์ reset รหัสผ่าน (ส่งไปที่ email) |
| **Reset password** | ตั้งรหัสผ่านใหม่ผ่านลิงก์จาก email |

### 2.2 Flow

- **Sign up:** กรอก email, password → ยืนยันอีเมล (ถ้าเปิด) → เข้าสู่ระบบ
- **Login:** กรอก email, password → เข้าสู่ระบบ
- **Continue with Google:** คลิกปุ่ม "Continue with Google" → เลือกบัญชี Google → เข้าสู่ระบบ
- **Forgot password:** กรอก email → ได้ลิงก์ reset ในอีเมล → คลิกลิงก์ไปหน้า Reset password
- **Reset password:** กรอกรหัสผ่านใหม่ 2 ช่อง → บันทึก → redirect ไป Login

### 2.3 หมายเหตุ

- **Google OAuth** ต้องมีใน Phase 1 — ตั้งค่าใน Supabase Dashboard (Google provider)
- Supabase รองรับ magic link, OAuth อื่นๆ (Apple, GitHub, etc.) — พิจารณาใน Phase 2
- Session management ใช้ Supabase เต็มรูปแบบ

## 4. ความชัดเจนเรื่องความถูกต้อง

### 4.1 นโยบายเมื่อผู้ใช้ขอเนื้อหาที่ผิดหลักวิชา

**คำตัดสิน:** แจ้งและเสนอทางเลือกที่ถูกต้องแทน

- ไม่ generate เนื้อหาที่ผิดหลักวิชา
- อธิบายว่าทำไมไม่เหมาะสม
- เสนอทางเลือกที่ถูกต้องให้ผู้ใช้เรียนต่อได้

**ตัวอย่างข้อความ:**
> "เรื่องนี้ไม่ตรงกับหลักวิชาการที่ยอมรับกันทั่วไป เราอยากช่วยคุณเรียนในทิศทางที่ถูกต้อง เช่น [ทางเลือก] ได้ไหม?"

### 4.2 ขอบเขตเนื้อหา (Scope)

**คำตัดสิน:** เฉพาะวิชาการ/ทักษะทั่วไป + ไม่อนุญาต medical, legal, financial advice

| อนุญาต | ไม่อนุญาต |
|--------|-----------|
| ภาษา, คณิต, วิทยาศาสตร์ | Medical advice |
| ประวัติศาสตร์, ภูมิศาสตร์ | Legal advice |
| ทักษะทั่วไป (การเขียน, การนำเสนอ) | Financial advice |
| วิชาการอื่นๆ | เนื้อหาอันตราย/ผิดกฎหมาย |

**ข้อยกเว้น:** ถ้าจำเป็นต้องมี medical/legal/financial ต้องมี disclaimer ชัดเจน

### 4.3 แหล่งอ้างอิง

**คำตัดสิน:** มี — แสดงแหล่งที่มาหลังจากทำ quiz/flashcard

- Phase 1: อาจเริ่มแบบง่าย เช่น "อ้างอิงจากหลักวิชาการที่ยอมรับกันทั่วไป"
- Phase 2: พัฒนาเป็นลิงก์/แหล่งอ้างอิงจริง

### 4.4 เรื่องที่ subjective

**คำตัดสิน:** อนุญาต — แต่ระบุชัดว่าเป็นความเห็นหรือการตีความ

- ไม่นำความเห็นมาใส่เป็นข้อเท็จจริง
- ใช้ข้อความแยก เช่น "นี่เป็นความเห็น/การตีความหนึ่ง ไม่ใช่ข้อเท็จจริงที่ตายตัว"

## 5. AI Guardrails + Fallback

### 5.1 ประเภทเนื้อหาที่ต้อง guard

**คำตัดสิน:** ทั้ง user prompt และ generated content

- **Input:** ป้องกัน prompt ที่ไม่เหมาะสม
- **Output:** ป้องกันเนื้อหาที่หลุดออกมา

### 5.2 สิ่งที่ต้อง block

| ประเภท | เหตุผล |
|--------|--------|
| เนื้อหาอันตราย/ความรุนแรง | ความปลอดภัย |
| เนื้อหาลามก | ความเหมาะสม |
| Medical/legal/financial advice | ความเสี่ยงต่อความเสียหาย |
| เนื้อหาที่ผิดหลักวิชาอย่างชัดเจน | หลักความถูกต้อง |
| Hate speech / การเลือกปฏิบัติ | ความปลอดภัยและความน่าเชื่อถือ |

### 5.3 เมื่อ AI block เนื้อหา

**คำตัดสิน:** แสดงข้อความที่เป็นมิตร ไม่เปิดเผยรายละเอียดทางเทคนิค

**ตัวอย่างข้อความ:**
> "เราไม่สามารถสร้างเนื้อหาตามที่ขอได้ในตอนนี้ อาจเป็นเพราะเนื้อหาไม่ตรงกับหลักวิชาการหรือไม่เหมาะสม ลองเปลี่ยนคำอธิบายหรือหัวข้อใหม่ได้นะ"

**ต้องมี:** ปุ่ม "ลองใหม่" หรือช่องให้แก้ prompt

### 5.4 Fallback เมื่อ AI error

**คำตัดสิน:** Retry อัตโนมัติ 2 ครั้ง → ถ้ายังไม่ได้ แสดง error message + ปุ่ม "ลองใหม่"

- ไม่ใช้ชุดคำถามสำเร็จรูปแทน (อาจไม่ตรงกับที่ผู้ใช้ขอ)
- ข้อความเมื่อ error: "เกิดข้อผิดพลาดชั่วคราว ลองกดปุ่มด้านล่างอีกครั้งนะ"

### 5.5 Retry logic

| รายการ | ค่า |
|--------|-----|
| จำนวน retry | 2 ครั้ง |
| Timeout | 30–45 วินาที |

## 6. สรุปตารางตัดสินใจ

| # | หัวข้อ | คำตัดสิน |
|---|--------|----------|
| 1 | นโยบายเนื้อหาผิดหลักวิชา | แจ้งและเสนอทางเลือกที่ถูกต้อง |
| 2 | ขอบเขตเนื้อหา | วิชาการ/ทักษะทั่วไป, ไม่รวม medical/legal/financial |
| 3 | แหล่งอ้างอิง | มี (Phase 1 ทำแบบง่ายก่อน) |
| 4 | เรื่อง subjective | อนุญาต แต่ระบุว่าเป็นความเห็น |
| 5 | AI provider | Anthropic |
| 6 | Guard อะไร | ทั้ง prompt และ generated content |
| 7 | Block อะไร | อันตราย, ลามก, medical/legal/financial, ผิดหลักวิชา, hate speech |
| 8 | เมื่อ AI block | ข้อความที่เป็นมิตร + ให้ลองใหม่ |
| 9 | Fallback | Retry 2 ครั้ง → error message + ปุ่ม "ลองใหม่" |
| 10 | Retry กี่ครั้ง | 2 ครั้ง |
| 11 | Timeout | 30–45 วินาที |
